# AGENTS.md — 기계 계약 (에이전트 실행 규칙)

이 파일은 이 저장소에서 작업하는 모든 코딩 에이전트의 **단일 계약**이다.
사람용 설명은 README.md, 절차는 docs/runbook-*, 이 파일은 규칙만 담는다.

## 0. 사명과 우선순위
한국 주식 프로그램 매매 시스템. 우선순위는 항상:
**① 돈을 잃지 않기(검증·리스크) > ② 정직한 표기 > ③ 수익 기능 추가 > ④ 코드 미관**

## 1. 지시 우선순위
1. 사용자의 명시적 지시
2. 이 파일(AGENTS.md)
3. docs/quality-gates.md (검증 기준)
4. docs/decisions/ 의 기존 결정 (뒤집으려면 새 결정 기록 필수)
5. README/기타 문서

## 2. 절대 규칙 (위반 금지)
- **검증 없는 수익 주장 금지.** 화면·문서의 모든 성과 수치는 (a) 재현 가능한 백테스트 스크립트와
  (b) 데이터 출처(생존/PIT 패널 구분)를 명시해야 한다.
- **전략 채택은 PIT 패널 검증 통과가 전제.** 생존 패널 단독 결과로 전략을 화면의 매수 신호로 올리지 않는다.
- **백테스트 필수 요건**: 룩어헤드 차단(신호=종가, 집행=익일 시가; 주봉=완성주만), 왕복 0.35% 비용,
  날짜군집 보정 통계(짝지은 차이·비중첩), 생존편향 고지. 기준: docs/quality-gates.md
- **.env(API 키)·data/cache(생성물)·state/(산출물)는 커밋 금지.** .gitignore 준수.
- **화면 통계 하드코딩 금지** — 빌드 시 원데이터에서 재계산해 임베드한다.
- 서버의 기존 서비스(nginx의 다른 도메인, 기존 cron)를 깨는 변경 금지. nginx 수정 시 백업+`nginx -t` 필수.

## 3. 작업 프로토콜
- **착수 전**: config/strategy.yaml(파라미터 단일 진실원)과 관련 docs/decisions/ 확인.
- **전략/신호 변경**: ① backtest/에 검증 스크립트 작성·실행 → ② docs/validation-status.md 갱신 →
  ③ 통과 시에만 app/·템플릿 반영 → ④ docs/decisions/에 결정 기록.
- **화면 변경**: 로컬 빌드(`python -m app.batch.build_webapp --out state/trade.html`) →
  렌더 검증(playwright 또는 payload 파싱) → 서버 배포 → 라이브 확인. 한 번에 한 단계.
- **데이터 파이프라인 변경**: 증분(update_data)·전종목(build_broad)·PIT(build_pit) 셋의 스키마
  정합성을 함께 확인. parquet 스키마 변경 시 서버 재백필 비용을 보고에 명시.
- **배포**: 서버 코드 동기화(sftp) 후 반드시 서버에서 빌드 1회 실행해 확인. cron이 다음날 같은
  코드로 돌기 때문에, 로컬만 고치고 서버를 안 올리면 다음 배치가 구버전으로 덮는다.

## 4. 검증 의무 (Definition of Done)
작업은 다음을 모두 만족해야 완료다:
- [ ] 코드 변경분이 실제 데이터로 1회 이상 end-to-end 실행됨
- [ ] 화면 변경이면 렌더링 확인(렌더된 DOM/스크린샷), 수치 변경이면 독립 재계산 대조
- [ ] 새 성과 주장이 있으면 quality-gates 충족 + validation-status 갱신
- [ ] 서버 반영이 필요한 변경이면 배포·라이브 확인까지
- [ ] 구조적 결정이면 docs/decisions/NNNN-*.md 기록

## 5. 중지·확대 조건
- 백테스트 결과가 기존 화면 표기와 모순되면 **즉시 표기 수정이 우선**, 신규 기능은 중단.
- 실거래 주문 코드 작성은 **사용자 명시 승인 전 금지** (현재 단계: 신호 생성 + 페이퍼 검증).
- 데이터 출처가 불확실한 수치를 발견하면 사용 중단하고 재계산 또는 사용자 보고.
- 같은 접근 2회 실패 시 접근을 바꾸거나 사용자에게 보고(무한 재시도 금지).

## 6. 비용·자원 한도
- KRX API: 일별 전종목 1콜/시장. 대량 백필은 백그라운드 + 중간 저장(재개 가능) 필수.
- OpenDART: 일 20,000건 한도 — 후보 종목(수십 개)에만 호출, 전종목 루프 금지.
- 서버 빌드는 cron(08:10)과 겹치지 않게 08:05~08:25 사이 수동 빌드 자제.

## 7. 컨벤션
- Python: 기존 스타일 유지(한국어 docstring/주석, 함수형 파이프라인). 새 의존성은 requirements.txt에 핀.
- 커밋: conventional commit(feat/fix/docs/chore) + 본문에 검증 방법 한 줄.
- 브랜치: main 직커밋 허용(단독 운영). 위험 변경은 브랜치+검증 후 머지.
- 테스트: 신호 로직 변경 시 tests/ 갱신, `pytest` 통과.

## 8. 현재 단계와 다음 관문
- 단계: **신호 생성 + 검증 재정립** (실거래 아님)
- 진행 중 관문: 시가총액 기반 유니버스 PIT 재검증 → 통과 시 모멘텀 전략 복권,
  실패 시 전략 재설계. 결과는 docs/validation-status.md에 기록.
- 실거래 전 필수: 페이퍼 트레이딩 1개월+ 실적, 월 손실 서킷브레이커 구현, 사용자 승인.
